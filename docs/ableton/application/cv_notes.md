# CV tailoring notes for the MTG–Steinberg application

The call's candidate profile is a checklist — the CV should make each bullet
findable in under five seconds:

1. **Master's degree (completed or near)** — put degree, institution, and
   (expected) completion date at the top. If the degree is in progress, say
   the expected month explicitly.
2. **Research experience relevant to the topic** — a "Research & Projects"
   section led by *Session State Explorer v0* with the repo URL, one line of
   what it demonstrates ("typed DAW-state graphs, descriptor linkage,
   explainable recommendations, learned chain-prediction baseline"), and any
   prior research work (theses, publications, research assistantships).
3. **Machine learning and audio signal processing** — name the concrete
   things: frameworks, courses, projects with a sentence of substance each.
   Avoid skill-cloud lists.
4. **Strong Python (C/C++ a plus)** — evidence beats claims: repos, production
   code, libraries used seriously (the prototype covers pydantic, NetworkX,
   librosa, Streamlit, pytest, CI).
5. **Musical background** — a short dedicated section: instruments,
   production/composition credits, releases, live work. This is a
   differentiator at MTG, not a footnote.
6. **English** — certificates or simply evidence (papers, talks, studies in
   English).
7. **Interdisciplinary/multicultural environments** — one line each for any
   cross-domain or international team experience.
8. **MTG values / responsible AI** — if you have open-source contributions,
   open datasets, or community work (e.g., Freesound uploads), list them; this
   maps directly to the call's open-science language.

Practical notes:
- One to two pages, reverse-chronological, PDF.
- Same header identity (name, email, GitHub) as the motivation letter.
- Link the demo video once it exists (Section 6 of the project plan).
