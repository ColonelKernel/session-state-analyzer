# Cubase adapter fixture bundle

Frozen v0.2 canonical bundle produced by the Cubase adapter
(`CubaseSessionStateExplorer`, branch `feat/canonical-export`):

```
cubase-explorer export-canonical fixtures/cubase/demo_session.dawproject \
    --out exports/demo_session
```

## Provenance

- **Adapter:** `cubase-session-state-explorer` 0.1.0 (`adapter_id:
  cubase-hybrid`); canonical-export mapper relocated from this repo's
  `session_explorer.drivers.cubase.mapper` at `SessionStateExplorer@041f529`.
- **Input:** `fixtures/cubase/demo_session.dawproject`
  (sha256 `78fa9f42fda02b5465f707cd004df9ab529b614abdfdcfd30b178eacde600a9f`),
  a *synthetic* DAWproject file generated by the Cubase repo's
  `tools/make_fixtures.py`, modeled on Cubase 15 exports (application tag
  "Cubase 15.0.10"). It was **not** produced by a running Cubase instance —
  the capability manifest says so in `notes` and leaves `tested_daw_version`
  unset.
- **Capture pathway:** `dawproject_export`, stability `OFFICIAL_EXPORT`
  (DAWproject is an open interchange format Cubase 14/15 import and export).
- **Sanitization:** exported with `sanitize=True`; home-directory and
  input-directory path prefixes are scrubbed from every string.
- **Determinism:** id counters reset per export; `created_at` is the input
  file's mtime; `snapshot_id` embeds the input content hash. Re-exporting the
  same input byte-for-byte reproduces `canonical.snapshot.json` and
  `native.json`.

| `adapter_descriptor.json` | identity card: modes, honest limitations |

| `canonical.snapshot.json` | flat v0.2 `CanonicalDAWSnapshot` |
| `validation.json` | `validate_snapshot()` report (valid, 0 errors/warnings) |
